## Overview

FinFram Screener supports any AI model that provides an OpenAI-compatible chat completions API. This includes models from Groq, Together.ai, OpenAI, and other compatible providers.

## API Compatibility Requirements

<Warning>
  **OpenAI-Compatible API Required**
  
  Models must support the OpenAI chat completions format. Some models like ChatGPT or Gemini might not work if their API structure differs.
</Warning>

Your API endpoint must:

<Steps>
  <Step title="Accept standard request format">
    ```json
    {
      "model": "model-name",
      "messages": [
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "User message"}
      ]
    }
    ```
  </Step>
  
  <Step title="Return standard response format">
    ```json
    {
      "choices": [
        {
          "message": {
            "content": "Assistant response"
          }
        }
      ]
    }
    ```
  </Step>
  
  <Step title="Support Bearer token authentication">
    ```
    Authorization: Bearer YOUR_API_KEY
    ```
  </Step>
</Steps>

## Model Configuration

Models are configured in `settings.json` with three components:

1. **Model identifier** - The specific model to use
2. **API endpoint** - The provider's chat completions URL
3. **Custom parameters** - Model-specific settings

## Model Parameters

The `custom-json` field allows you to configure model-specific parameters. These are merged with the base request payload in `src/groq_api.py:21`:

```python src/groq_api.py
response = requests.post(url, headers=headers, json=payload | SETTINGS["custom-json"])
```

### Common Parameters

<ParamField path="temperature" type="float" default="0.7">
  Controls randomness in responses (0.0 - 2.0)
  
  - **Lower values (0.1-0.3)**: More focused and deterministic
  - **Medium values (0.5-0.7)**: Balanced creativity and consistency
  - **Higher values (0.8-1.0)**: More creative and varied
  
  <Info>
    For financial analysis, use lower temperatures (0.3-0.5) for more consistent, fact-based responses.
  </Info>
</ParamField>

<ParamField path="max_tokens" type="integer" default="varies">
  Maximum length of the generated response
  
  - **512-1024**: Short summaries and quick insights
  - **1024-2048**: Detailed analysis
  - **2048-4096**: Comprehensive reports
  
  <Warning>
    Higher token limits increase API costs. Adjust based on your needs.
  </Warning>
</ParamField>

<ParamField path="top_p" type="float" default="1.0">
  Nucleus sampling parameter (0.0 - 1.0)
  
  Controls diversity by considering tokens with cumulative probability `top_p`.
  
  - **0.1**: Very focused, deterministic
  - **0.9**: Balanced
  - **1.0**: Maximum diversity
</ParamField>

<ParamField path="frequency_penalty" type="float" default="0.0">
  Reduces repetition of tokens (-2.0 to 2.0)
  
  - **Positive values**: Penalize repeated phrases
  - **0.0**: No penalty
  - **Negative values**: Encourage repetition
</ParamField>

<ParamField path="presence_penalty" type="float" default="0.0">
  Encourages new topics (-2.0 to 2.0)
  
  - **Positive values**: Encourage new topics
  - **0.0**: No penalty
  - **Negative values**: Encourage staying on topic
</ParamField>

## Switching Between Models

To switch between different models or providers:

<Steps>
  <Step title="Update settings.json">
    Modify the `model` and `url` fields in `settings.json`:
    
    ```json settings.json
    {
        "model": "new-model-identifier",
        "url": "https://new-provider.com/v1/chat/completions",
        "custom-json": {
            "temperature": 0.7
        }
    }
    ```
  </Step>
  
  <Step title="Update API key if needed">
    If switching providers, update your `.env` file with the new API key:
    
    ```env .env
    API_KEY=new_provider_api_key
    ```
  </Step>
  
  <Step title="Test the configuration">
    Run the screener to verify the new model works:
    
    ```bash
    python main.py
    ```
  </Step>
</Steps>

<Note>
  No code changes are required to switch models - just update `settings.json` and optionally `.env`.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Model not found error">
    **Cause:** Model identifier doesn't match the provider's format.
    
    **Solution:**
    - Check the provider's documentation for exact model names
    - Verify the model is available in your region/tier
    - Groq models often use `openai/` prefix
    - Together.ai models use full paths like `meta-llama/...`
  </Accordion>
  
  <Accordion title="Invalid parameter errors">
    **Cause:** The model doesn't support a parameter in `custom-json`.
    
    **Solution:**
    - Remove unsupported parameters from `custom-json`
    - Check the provider's documentation for supported parameters
    - Start with minimal config (only `temperature`)
  </Accordion>
  
  <Accordion title="Response format errors">
    **Cause:** API doesn't follow OpenAI format.
    
    **Solution:**
    - Verify the endpoint returns `{"choices": [{"message": {"content": "..."}}]}`
    - Test with a simple curl request first
    - Check if the provider offers an OpenAI-compatible endpoint
  </Accordion>
  
  <Accordion title="Slow or timeout errors">
    **Cause:** Model is too large or provider is slow.
    
    **Solution:**
    - Reduce `max_tokens` to get faster responses
    - Switch to a smaller/faster model
    - Use Groq for fastest inference
  </Accordion>
</AccordionGroup>
