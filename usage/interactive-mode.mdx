## Overview

After the initial stock analysis, FinFram Screener automatically enters **Interactive Mode**, allowing you to ask unlimited follow-up questions about the stock without re-running the analysis.

## How Interactive Mode Works

Once the initial forecast is displayed, you'll see this prompt:

```
⤷ Enter your prompt:
```

The system is now waiting for your input. You can:
- Ask specific questions about the analysis
- Request deeper insights on particular metrics
- Compare scenarios
- Explore risks and opportunities in detail

## Implementation Details

Interactive mode is powered by a continuous loop in the main application:

**Source:** `main.py:18-22`
```python
while True:
    console.print("⤷ Enter your prompt:", style="bold red")
    prompt = input()
    result = query_api(prompt)
    display_result(result)
```

<Info>
The loop runs indefinitely, allowing unlimited follow-up questions. Each response is formatted and displayed using Rich markdown rendering.
</Info>

## Prompt Engineering Tips

### Use Action Verbs
```
✅ "Explain the significance of the price volatility"
✅ "Compare this stock to its competitors"
✅ "Identify potential catalysts for growth"
```

### Provide Context
```
✅ "As a risk-averse investor, should I be concerned about the volatility?"
✅ "For a 5-year investment horizon, what are the key considerations?"
```

### Ask for Formats
```
✅ "List the top 5 risks in order of severity"
✅ "Summarize the investment thesis in 3 bullet points"
```

## How to Exit Interactive Mode

To exit the interactive mode and end the screener session:

<Steps>
  <Step title="Keyboard Interrupt">
    Press `Ctrl+C` (or `Cmd+C` on Mac) in the terminal.
    
    This sends a keyboard interrupt signal that terminates the Python process.
  </Step>
  
  <Step title="Close Terminal">
    Alternatively, simply close the terminal window to end the session.
  </Step>
</Steps>

<Warning>
There is no "exit" command built into the application. You must use keyboard interrupt to stop the program.
</Warning>

## Understanding the Response Flow

Each time you enter a prompt:

<Steps>
  <Step title="Prompt Submission">
    Your question is captured via `input()` and sent to the AI.
  </Step>
  
  <Step title="API Processing">
    The `query_api()` function sends your prompt to Groq's API with the configured model.
    
    **Source:** `groq_api.py:10-26`
  </Step>
  
  <Step title="Response Display">
    The AI's response is rendered in markdown format with syntax highlighting.
    
    **Source:** `utils.py:4-7`
  </Step>
  
  <Step title="Next Prompt">
    The loop continues, ready for your next question.
  </Step>
</Steps>

## Technical Notes

### Session Persistence
<Info>
Each follow-up question is sent independently to the API. The system does not maintain conversation history across prompts within the same session.
</Info>

If you need context from previous responses, include that information in your new prompt:

```
You mentioned high volatility earlier. What timeframe shows this best?
```

### API Call Efficiency

Every prompt in interactive mode triggers a new API call:
- **Initial analysis**: 1 API call
- **Each follow-up**: 1 additional API call

<Warning>
Be mindful of API rate limits if asking many questions in quick succession.
</Warning>
