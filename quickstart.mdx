# Quick Start

Get your first stock analysis running in minutes.

## Configuration

<Steps>
  <Step title="Configure AI model settings">
    Open `settings.json` in the root directory and configure your AI model:
    
    <CodeGroup>
    ```json settings.json
    {
        "model": "openai/gpt-oss-120b",
        "url": "https://api.groq.com/openai/v1/chat/completions",
        "custom-json": {
            "temperature": 0.7
        }
    }
    ```
    </CodeGroup>
    
    **Configuration options:**
    - `model` - The AI model identifier (e.g., `openai/gpt-oss-120b`, `mixtral-8x7b-32768`)
    - `url` - OpenAI-compatible API endpoint URL
    - `custom-json` - Additional model-specific parameters like temperature, max_tokens, etc.
    
    <Note>
      FinFram Screener works with any OpenAI-compatible API. Popular providers include Groq, Together.ai, and OpenRouter.
    </Note>
    
    <Warning>
      Some AI models (like ChatGPT or Gemini) might not work if their API structure differs from the OpenAI standard.
    </Warning>
  </Step>
  
  <Step title="Create .env file with API key">
    Create a `.env` file in the root directory and add your API key:
    
    ```env .env
    API_KEY=your_api_token_here
    ```
    
    Replace `your_api_token_here` with your actual API key from your chosen provider (Groq, Together.ai, etc.).
    
    <Warning>
      Never commit your `.env` file to version control. It contains sensitive credentials.
    </Warning>
  </Step>
  
  <Step title="Run the screener">
    Start the FinFram Screener:
    
    ```bash
    python main.py
    ```
    
    The application will launch and display the welcome message.
  </Step>
  
  <Step title="Enter a stock ticker">
    When prompted, enter a stock ticker symbol (e.g., AAPL, TSLA, MSFT):
    
    ```
    📊 Welcome to FinFram Screener
    Enter stock ticker (e.g., AAPL): AAPL
    ```
    
    The screener will:
    1. Fetch 6 months of historical data for the ticker
    2. Compute price statistics and volatility metrics
    3. Build a structured prompt with the data
    4. Query the AI model for analysis and forecasting
    5. Display the AI-generated insights
  </Step>
  
  <Step title="Interactive analysis">
    After the initial forecast, you can continue the conversation with follow-up questions:
    
    ```
    ⤷ Enter your prompt:
    What are the main risks for this stock?
    ```
    
    The screener maintains context and allows interactive exploration of the analysis.
  </Step>
</Steps>

## Under the Hood

Here's the main flow from `main.py`:

<CodeGroup>
```python main.py
from stock_data import get_stock_history
from prompt_builder import build_prompt
from groq_api import query_api
from utils import display_result
from rich.console import Console

def main():
    console = Console()

    print("📊 Welcome to FinFram Screener")
    ticker = input("Enter stock ticker (e.g., AAPL): ").upper()
    history = get_stock_history(ticker)

    if history is not None:
        finance = build_prompt(ticker, history)
        result = query_api(finance)
        display_result(result)
        while True:
            console.print("⤷ Enter your prompt:", style="bold red")
            prompt = input()
            result = query_api(prompt)
            display_result(result)
    else:
        print("❌ Could not retrieve stock data. Please try another ticker.")

if __name__ == "__main__":
    try:
        main()
    except Exception as e:
        print(f"❌ Error: {e}")
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Stock data not found">
    If you see "Could not retrieve stock data", verify:
    - The ticker symbol is correct and exists on Yahoo Finance
    - You have an active internet connection
    - yfinance is properly installed
  </Accordion>
  
  <Accordion title="API errors">
    If you encounter API errors:
    - Verify your API_KEY is correct in the .env file
    - Check that the API endpoint URL in settings.json is valid
    - Ensure your API key has sufficient credits/quota
  </Accordion>
  
  <Accordion title="Module import errors">
    If you see import errors:
    - Ensure you're running from the correct directory
    - Verify all dependencies are installed: `pip install -r requirements.txt`
    - Check that you're using Python 3.7 or higher
  </Accordion>
</AccordionGroup>
